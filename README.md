# 产品详细设计文档 Skill

## 概述

这是一个专为产品经理设计的"去技术化"详细设计文档生成工具，聚焦于**业务逻辑、交互行为与数据规则**的定义，而非底层代码实现。

## 核心理念

> **本规范的核心目的在于确立一种"去技术化"的详细设计文档模式**，要求产品经理聚焦于业务逻辑（Business Logic）、**交互行为（Interaction）与数据规则（Data Rules）**的定义，而非底层代码实现。通过标准化的文档架构，消除需求文档（PRD）与开发实现之间的模糊地带，确保研发团队准确理解业务意图，提升交付质量。

## 功能特性

- ✅ **业务逻辑优先** - 定义业务对象、状态流转、业务规则
- ✅ **交互行为清晰** - BPMN 2.0流程图、界面交互规则
- ✅ **数据规则明确** - 业务对象关系、数据约束、交换规则
- ✅ **完全去技术化** - 无编程语法、无数据库设计、无API接口定义
- ✅ **标准化结构** - 7章完整文档框架
- ✅ **Markdown格式** - 版本控制友好

## 适用场景

本文档模板适用于：
- 产品经理编写详细设计文档
- 产品负责人规划功能模块
- 业务分析师梳理业务流程
- 需求评审与开发交接

## 文档结构

详细设计文档包含以下7个主要部分：

### 1️⃣ 文档概览
- 项目背景与目标
- 范围与边界
- 术语与定义

### 2️⃣ 系统功能架构
- 业务模块关系（而非技术分层）
- 角色与权限

### 3️⃣ 核心业务对象
- 业务对象关系（ERD图，去PK/FK技术标记）
- 业务状态流转（状态机设计）

### 4️⃣ 功能模块设计
- 业务流程设计（BPMN 2.0）
- 界面与交互
- 业务处理步骤（非伪代码）

### 5️⃣ 数据交换需求
- 与其他系统的数据交换
- 数据同步规则

### 6️⃣ 业务约束与体验要求
- 用户体验要求
- 业务规则约束
- 安全与合规

### 7️⃣ 附录
- 参考文献
- 相关文档

## 与技术文档的区别

| 维度 | 本文档（产品详细设计） | 技术架构文档（TDD） |
|------|---------------------|-------------------|
| **目标读者** | 产品经理、业务分析师、开发人员 | 架构师、技术负责人 |
| **核心内容** | 业务逻辑、交互行为、数据规则 | 技术架构、数据库设计、API定义 |
| **描述方式** | 业务语言、表格、流程图 | 技术语言、类图、时序图 |
| **状态机设计** | 业务状态定义 + 流转规则 | 状态模式实现 + 代码结构 |
| **对象模型** | 业务对象关系 + 属性说明 | ERD图 + 表结构 + 索引 |
| **流程描述** | BPMN 2.0流程图 + 步骤说明 | 时序图 + 活动图 |
| **数据交换** | 数据项 + 对应关系 | REST API + JSON Schema |
| **约束说明** | 业务规则 + 用户体验 | 性能指标 + 并发数 |

## 设计原则

### 去技术化原则
- ❌ 不使用编程语言语法（Java/C#/Python等）
- ❌ 不包含数据库表结构设计
- ❌ 不定义API接口（REST/GraphQL等）
- ❌ 不涉及技术架构选型

### 业务聚焦原则
- ✅ 使用业务语言和表格描述
- ✅ 定义业务对象的状态流转
- ✅ 描述用户交互行为
- ✅ 明确数据规则和约束

### 标准化原则
- ✅ 统一的文档结构
- ✅ 规范的术语定义
- ✅ 明确的验收标准
- ✅ 统一的排版格式

## 文档格式要求

> **所有详细设计文档必须遵从以下排版参数，以确保打印与归档的统一性。**

### 页面设置

- **纸张大小**：A4 标准纸
- **页边距**：左右边距 3.18cm，上下边距 2.54cm
- **页码**：页脚居中显示

### 正文格式

- **字体**：宋体，小四号（12pt）
- **行距**：1.5倍
- **首行缩进**：正文段落首行缩进 2 字符
- **英文字体**：Times New Roman（用于数字、英文单词）

### 标题分级标准

文档标题严格执行以下分级标准：

| 标题级别 | 字号 | 格式 | 序号格式 | 示例 |
|---------|------|------|----------|------|
| 一级标题 | 二号（22pt） | 加粗 | 中文数字"一、" | 一、文档概览 |
| 二级标题 | 小二（18pt） | 加粗 | 中文数字加括号"（一）" | （一）项目背景 |
| 三级标题 | 三号（16pt） | 加粗 | 阿拉伯数字"1." | 1. 业务背景 |
| 四级标题 | 小四（12pt） | 加粗 | 阿拉伯数字加括号"（1）" | （1）业务痛点 |

### Markdown格式转换说明

本模板使用 Markdown 格式编写以支持版本控制，导出为 Word/PDF 时需按上述格式调整：
- Markdown的 `#` 对应一级标题"一、"
- Markdown的 `##` 对应二级标题"（一）"
- Markdown的 `###` 对应三级标题"1."
- Markdown的 `####` 对应四级标题"（1）"
