# 产品详细设计文档生成 Skill

## Skill 概述

本 Skill 帮助用户生成符合业界标准的产品详细设计文档，基于多个GitHub优秀开源项目的最佳实践。

## 使用方法

在 Claude Code 中调用此 Skill：

```
请帮我生成产品详细设计文档
```

## 交互流程

1. **收集项目基本信息**
   - 项目名称和描述
   - 项目背景和目标
   - 技术栈偏好
   - 团队规模和角色

2. **确定文档范围**
   - 需要包含的架构视图（应用/开发/安全/规模/基础设施）
   - 文档详细程度
   - 特殊关注点

3. **生成定制化文档**
   - 基于模板生成详细设计文档
   - 根据项目特点调整内容
   - 补充具体的技术细节

4. **输出文档**
   - Markdown 格式文档
   - 可直接用于项目文档
   - 支持 Git 版本控制

## 功能特性

### 全面的文档结构

- **5大架构视图**：应用、开发、安全、规模、基础设施
- **标准化格式**：遵循 IEEE 1016-2009 和 ISO/IEC/IEEE 42010 标准
- **完整的内容**：涵盖功能设计、接口设计、数据设计、架构决策等

### 灵活的定制能力

- 根据项目规模调整详细程度
- 支持不同架构风格（单体、微服务、Serverless等）
- 适配不同技术栈

### 实用的辅助功能

- 架构决策记录（ADR）模板
- 非功能性需求检查清单
- 性能指标和容量规划指南
- 安全合规检查项

## 最佳实践建议

### 文档编写原则

1. **保持简洁** - 每个字都有意义，避免冗余
2. **持续更新** - 文档随项目演进而更新
3. **版本控制** - 使用 Git 管理文档版本
4. **图文并茂** - 使用图表增强可读性
5. **可追溯性** - 建立需求-设计-决策的追溯链

### 不同项目规模的建议

| 项目规模 | 详细程度 | 重点章节 |
|---------|---------|----------|
| 小型项目（1-3人） | 简化 | 应用视图、开发视图 |
| 中型项目（4-10人） | 标准 | 全部视图 |
| 大型项目（10+人） | 详细 | 全部视图 + 详细补充 |

### 文档维护建议

- **定期审查**：每个迭代周期审查一次
- **版本标记**：重要决策添加版本标记
- **变更记录**：记录文档的变更历史
- **同步更新**：代码变更时同步更新文档

## 常见问题

**Q: 是否必须填写所有章节？**
A: 不需要。根据项目实际情况，不适用的章节可以标注"N/A"。

**Q: 文档应该在什么时候编写？**
A: 建议在设计阶段开始编写，在开发过程中持续完善。

**Q: 如何处理文档更新？**
A: 使用 Git 进行版本控制，每次重大变更更新修订历史。

**Q: 文档需要多详细？**
A: 根据项目规模和团队需求决定。避免过度设计，保持适度详细。
